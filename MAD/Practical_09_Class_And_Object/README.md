# Practical 9: Class and Object

## Introduction to Object-Oriented Programming (OOP)
This is where programming becomes incredibly powerful because we start designing things that mirror the real world!

### What is a Class?
A **Class** is a blueprint. Imagine an architect drawing the plans for a house. The paper plan isn't a physical house, but it describes exactly what the house will have: 3 bedrooms, red paint, a front door.
In our code, we made a `Class` called `Car`. It dictates that ANY car we create will have a `brand`, a `color`, and the ability to `startEngine()`.

### What is an Object?
An **Object** is the real thing built from the blueprint. 
Following the architect analogy, the Object is the actual house where you can go inside and live.
In our code, `myDreamCar` is the object! We took the theoretical blueprint of a `Car`, birthed a fast, real "Tesla" that is "Red", and physically started its engine.

Classes let us create as many real objects as we want cleanly!
